# 注釈ワークフローガイド

## はじめに

このドキュメントでは、Marginaliaの注釈機能の使い方を説明します。

## 注釈の種類

Marginaliaでは4種類の注釈をサポートしています:

### 1. コメント (Comment)

一般的なコメントや気づきを記録します。

例: この部分の説明がわかりにくいかもしれません。

### 2. 校閲 (Review)

レビューや修正提案を記録します。

例: この文章は「である調」に統一すべきです。

### 3. 保留 (Pending)

後で確認が必要な項目をマークします。

例: この数値は最新のデータで確認が必要です。

### 4. 議論 (Discussion)

チームで議論が必要な項目をマークします。

例: この機能の優先度について話し合いが必要です。

## 注釈の追加方法

1. プレビューまたはエディタでテキストを選択
2. ポップアップメニューから注釈タイプを選択
3. コメント内容を入力
4. 「追加」をクリック

## 注釈の管理

### フィルタリング

注釈パネルでは以下のフィルタが利用できます:

- **ステータス**: すべて / 未解決 / 解決済み
- **タイプ**: コメント / 校閲 / 保留 / 議論

### 解決

注釈の問題が解決したら、「解決」ボタンをクリックして解決済みにマークします。

### 返信

注釈に対して返信を追加することで、議論を続けることができます。

## ベストプラクティス

1. **明確で具体的なコメント**を心がける
2. **適切なタイプ**を選択する
3. **解決したら即座にマーク**する
4. **定期的にフィルタで確認**する

## ショートカット

| 操作 | ショートカット |
|------|----------------|
| 保存 | Ctrl/Cmd + S |
| 元に戻す | Ctrl/Cmd + Z |
| やり直す | Ctrl/Cmd + Y |
